# Brows

Основа кастомного десктопного браузера на Electron: своя рамка окна, вкладки,
адресная строка с переключением поисковика (Google / Яндекс) и панель
визуальной настройки — акцентный цвет, цвета шапки, скругления, плотность
интерфейса, шрифт и фон страницы новой вкладки. Всё это без пересборки —
меняется вживую и сохраняется на диске.

## Почему Electron, а не iframe-обёртка

Google и Яндекс запрещают встраивать свои страницы в `<iframe>` (заголовки
`X-Frame-Options` / CSP). Настоящий браузер должен сам управлять движком
страницы — поэтому здесь используется `<webview>` в Electron: у каждой
вкладки свой процесс рендеринга Chromium, как в настоящем браузере, а не
имитация в веб-странице.

## Запуск

```bash
npm install
npm start
```

Требуется Node.js 18+. При первом запуске Electron скачается автоматически
через `npm install` (нужен интернет).

## Структура проекта

```
brows/
├── main.js            # главный процесс: окно, IPC, хранение настроек в userData
├── preload.js          # безопасный мост между main и renderer (contextBridge)
├── package.json
└── src/
    ├── browser.html     # шапка окна, вкладки, тулбар, панель настроек
    ├── browser.css       # тема через CSS-переменные (--accent, --radius, ...)
    ├── browser.js        # логика вкладок, навигации, поиска, настроек
    ├── newtab.html        # страница новой вкладки
    ├── newtab.css
    └── newtab.js           # читает настройки темы из query-параметров URL
```

## Что уже реализовано

- **Своя рамка окна** (`frame: false`) — шапка, вкладки и кнопки свернуть /
  развернуть / закрыть нарисованы вручную и полностью стилизуемы.
- **Вкладки** — открытие/закрытие, переключение, заголовок и favicon
  подтягиваются из `webview`, `Ctrl+T` / `Ctrl+W` / `Ctrl+L`.
- **Адресная строка** — распознаёт URL и поисковый запрос; для запроса
  строится ссылка на Google или Яндекс в зависимости от выбранной системы
  (кнопка-переключатель `G` / `Я` прямо в адресной строке).
- **Навигация** — назад / вперёд / обновить / домой, индикатор загрузки.
- **Панель настроек** (шестерёнка в тулбаре):
  - выбор поисковика по умолчанию;
  - домашняя страница;
  - акцентный цвет (пресеты + произвольный `<input type="color">`);
  - цвет шапки и активной вкладки;
  - радиус скругления элементов (слайдер);
  - плотность интерфейса (просторно / компактно);
  - шрифт интерфейса (системный / моно / скруглённый);
  - фон страницы новой вкладки: градиент, сплошной цвет или картинка по URL;
  - сброс всех настроек.
- **Новая вкладка** — часы, поле поиска, сетка ярлыков; фон и акцент
  подхватываются из настроек через query-параметры URL (без лишнего IPC).
- **Настройки хранятся на диске** (`app.getPath('userData')`) и переживают
  перезапуск приложения.

## Что стоит добавить дальше (осознанно оставлено за скобками MVP)

- Редактор ярлыков новой вкладки прямо в UI (сейчас список задаётся в
  `DEFAULT_SETTINGS.newtab.shortcuts` в `main.js`, редактируется вручную).
- История посещений и полноценные закладки с UI.
- Загрузки файлов (`will-download` в `main.js`) и панель загрузок.
- Приватные вкладки (`webview partition="persist:..."` / без `persist:`).
- Менеджер тем — сохранение/экспорт нескольких профилей оформления.
- Автообновление и сборка установщиков через `electron-builder`.

## Идея кастомизации

Вся тема живёт в CSS-переменных на `:root` (`browser.css`) и применяется
функцией `applyTheme()` в `browser.js`: любое поле в `settings.theme`
мгновенно превращается в `--accent`, `--chrome-bg`, `--radius` и так далее —
без перезагрузки окна. Это тот же принцип, что и в дизайн-токенах:
один источник правды → визуальный результат.
